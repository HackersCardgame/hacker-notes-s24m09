

meine SUBJEKTIVE best practices um Fehler zu finden, egal ob Windows Server oder Linux


1| ist es nur ein Fehler, dann ist es einfach eine Normale Fehlersuche

2| sind es mehr als ein Fehler, sind es zwei Normale Fehlersuchen

2+| sind es mehr als ein Fehler, welche aber interagieren miteinander

BEISPIEL:
* also zB eine falsche Einstellung in der apache2.conf 
+ aber auch noch ein Problem mit dem Apache2 daemon selber 

=> dann wird es mühsam für junge Menschen diese Sorte von Fehler zu finden und zu reparieren
=> Fehler die miteinander inteargieren und allenfalls sogar im Sinne eines "deadlocks" (Wikipedia Arikel allenfalls lesen) verhindern dass man das System wieder repariert bekommt erfordern dann detailwissen sowohl über die apache2.conf als auch darüber wie man z.B. im Apache2 daemon Dinge repariert oder auch nur im Source Code nachliest weshalb jetzt spezifischer Fehler entsteht
